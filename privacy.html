Privacy Policy

Last updated: July 23, 2025

This Privacy Policy explains in detail how data is handled and what measures are in place to ensure your privacy when using the USCIS Case Tracker application, developed by Yaiquel Carmenates. Our commitment is to provide a service that respects your privacy fully.

1. Types of Data Collected

The USCIS Case Tracker application is designed to operate with minimal data interaction. We want to be absolutely clear: no personal data is collected, stored, or shared by our application. This includes, but is not limited to:

  • Geolocation Data: We do not access or store your location information.
  • Financial Information: We do not collect any financial details such as credit card numbers, bank accounts, or payment history.
  • Medical Information: We do not collect any health or medical-related data.
  • User Contacts: We do not access your device's contacts list.
  • Identifiers or Cookies: We do not use persistent identifiers, cookies, or any similar tracking technologies that could identify you personally or track your online activity.
  • Case Numbers: While you input USCIS case numbers into the app, these are processed temporarily and immediately discarded after the query to the public USCIS API is made. They are not stored or associated with you.

2. How Data is Used (or Not Used)

All queries initiated within the USCIS Case Tracker application are sent directly to the public USCIS API. The purpose of this application is solely to facilitate the direct retrieval of publicly available case status information. Our processing is designed to be stateless:

  • Case numbers you enter are used only for the purpose of making an immediate request to the USCIS API.
  • Immediately after receiving a response from the USCIS API, the case number is discarded from our system's temporary memory.
  • We do not engage in any form of data tracking, user profiling, or long-term storage of your queries or case numbers.
  • No de-identified, anonymized, or pseudonymized data is created or used, as no personal data is collected in the first place.

3. Third Parties and Data Sharing

We want to ensure absolute transparency regarding third-party involvement and data sharing:

  • No data is shared with any third parties whatsoever. This includes, but is not limited to, advertisers, marketing partners, data brokers, or any other entities.
  • The application does not integrate with any analytics services, advertising networks, or third-party tracking libraries.
  • The only "third party" involved is the official USCIS API, which is a public service for retrieving case status information. Your interaction is directly with this public API for the sole purpose of checking your case status.
  • Third parties are not bound by any terms or conditions in our privacy policy, because no data is shared with them.

4. Data Sale and Monetization

A core principle of USCIS Case Tracker is privacy and user focus:

  • User data is absolutely not sold, rented, or monetized in any form.
  • Our business model does not rely on selling or profiting from your personal information or usage data.

5. User Rights and Control

Given that no personal data is collected or stored, users maintain full control and transparency over their interaction with the application:

  • Users may stop using the application at any time. There will be no lingering data associated with them, as none was ever stored.
  • There are no user accounts, profiles, or settings to manage regarding data, as the application does not retain any user-specific information.
  • This design inherently protects your privacy by ensuring there is no personal data for us to manage or for you to request access to.

6. Breach Notification

While the design minimizes risks by not storing personal data, we have a clear protocol for any security incidents:

  • In the extremely unlikely event of a security breach that could potentially impact any aspect of user interaction or data (even temporary non-personal data), we commit to notifying users immediately.
  • Notification would occur through prominent in-app messages and updates on our official communication channels (e.g., website, app store description if applicable), providing clear instructions for any recommended actions.

7. Data Retention Policy

Our data retention policy is straightforward because of our design:

  • No user data is stored persistently. Each interaction with the USCIS API is stateless and anonymous.
  • There is no concept of "dormant" data, as data is never retained beyond the immediate processing of a single query.

8. Data Deletion

The principle of no data collection extends to deletion:

  • Since no personal data or usage history is retained by the application, there is nothing for users to delete.
  • Should future versions of the application ever require the collection or storage of any form of user data (which would only occur with explicit user consent and a revised policy), comprehensive data deletion options would be provided to users.

9. Transfer of Ownership

In the event of a change in ownership of the USCIS Case Tracker application:

  • Users will be prominently notified of any such change.
  • No user data will be transferred to the new entity. As no personal data is collected or stored, there is nothing to transfer.
  • The new owner would be required to adhere to a privacy policy that aligns with or exceeds the current privacy standards, and users would be provided with clear options regarding their continued use of the application under new terms.

10. Secure Data Disposal

Even for transient operational data, security is paramount:

  • Although no personal data is stored, any temporary system logs or operational data generated during the brief processing of an API query are handled with strict security measures.
  • Such temporary data, if it exists, is securely disposed of immediately after its intended use, utilizing industry-standard encryption and data sanitization protocols to prevent any recovery.
  • There is no user-downloadable data from our end, as we do not store it.

11. Account Management

The USCIS Case Tracker application does not require or offer user accounts:

  • There are no user accounts to create, manage, or close, eliminating the need for account-related data handling.
  • This design further reinforces user privacy by removing the necessity for personal authentication or profile creation.

12. Changes to This Policy

We are committed to transparency regarding any updates to our Privacy Policy:

  • Any future changes to this Privacy Policy will be clearly posted on this page, with a revised "Last updated" date at the top.
  • For any significant or material changes that might affect user interactions (e.g., if future features ever necessitated data collection), users would be notified proactively through prominent in-app messages and clear, plain-language summaries of what has changed.
  • Continued use of the application after such notifications would imply acceptance of the updated terms.

13. Consent

Given the current design of the USCIS Case Tracker application, which does not collect or store any personal data, explicit consent for data processing is not required.

  • There is no personal data for which to obtain consent, as our operations are entirely anonymous and stateless regarding user information.
  • Should any future version of the application introduce features that necessitate the collection of any form of personal data, explicit and informed user consent would be unequivocally required before such data collection begins. This would involve clear opt-in mechanisms and a detailed explanation of the data to be collected, its purpose, and how it will be used.
